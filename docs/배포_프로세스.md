# 🚀 자동 배포 프로세스

> Cafe24 서버에 코드가 어떻게 자동으로 반영되는지 설명합니다.

---

## 배포 아키텍처

```
GitHub Repository (main 브랜치)
         │
         │ push / merge 이벤트 발생 시 자동 트리거
         ▼
GitHub Actions Runner (ubuntu-latest)
         │
         │ 1. 저장소 코드 체크아웃
         │ 2. FTP-Deploy-Action 실행
         ▼
Cafe24 FTP 서버 (gpchurch.mycafe24.com:21)
         │
         │ 변경된 파일만 업로드 (증분 배포)
         ▼
/www/wp-content/themes/
├── gapyeong-church/         ← 메인 테마 업로드
└── gapyeong-church-child/   ← 자식 테마 업로드
         │
         ▼
https://gpchurch.mycafe24.com  ✅ 반영 완료
```

---

## 배포 트리거 조건

`main` 브랜치에 아래 이벤트가 발생할 때 자동 실행됩니다:

| 이벤트 | 설명 |
|--------|------|
| `push` | main 브랜치에 직접 Push |
| PR Merge | Pull Request가 main으로 병합 |
| `workflow_dispatch` | GitHub Actions 탭에서 수동 실행 |

---

## 배포 소요 시간

| 단계 | 소요 시간 |
|------|----------|
| Actions 시작 대기 | ~10초 |
| 코드 체크아웃 | ~10초 |
| 파일 비교 | ~20초 |
| FTP 업로드 (첫 배포) | ~2~5분 |
| FTP 업로드 (변경분만) | ~30초~1분 |

---

## 배포 확인 방법

1. https://github.com/touchxe/gpchurch/actions 접속
2. 최근 실행된 워크플로 확인

**상태 아이콘 의미:**

| 아이콘 | 의미 |
|--------|------|
| 🟡 주황 원 | 실행 중 |
| ✅ 초록 체크 | 성공 |
| ❌ 빨간 X | 실패 |

---

## 배포 실패 시 대처

1. 실패한 워크플로 클릭
2. 실패한 Step 클릭하여 로그 확인
3. YoungBin에게 오류 내용 전달

**자주 발생하는 오류:**

| 오류 | 원인 | 해결 |
|------|------|------|
| FTP 연결 실패 | 서버 점검 중 | 잠시 후 재시도 |
| 권한 오류 | FTP 비밀번호 변경 | YoungBin이 Secrets 업데이트 |
| 파일 없음 | 경로 오류 | YoungBin이 YAML 수정 |

---

## GitHub Secrets 관리

배포에 사용되는 민감 정보는 GitHub Secrets에 안전하게 보관됩니다.  
**관리자(YoungBin)만 접근/수정 가능합니다.**

| Secret 이름 | 용도 |
|------------|------|
| `FTP_HOST` | Cafe24 FTP 서버 주소 |
| `FTP_USER` | FTP 사용자 이름 |
| `FTP_PASS` | FTP 비밀번호 |
| `SSH_PRIVATE_KEY` | SSH 개인 키 (백업용) |
