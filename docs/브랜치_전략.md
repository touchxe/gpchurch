# 🌿 브랜치 전략

> 이 프로젝트의 Git 브랜치 관리 규칙입니다.

---

## 브랜치 구조

```
main (보호된 브랜치 - 라이브 서버와 연동)
  │
  ├── YoungBin    ← 관리자 작업 브랜치
  ├── HyunChan    ← HyunChan 작업 브랜치
  └── JunSeok     ← JunSeok 작업 브랜치
```

---

## 브랜치 규칙

### `main` 브랜치
- **직접 Push 금지** (YoungBin만 긴급 시 허용)
- PR Merge를 통해서만 코드 반영
- 이 브랜치에 변경이 생기면 **자동으로 서버에 배포됨**

### 개인 브랜치 (HyunChan, JunSeok 등)
- 각자 자신의 브랜치에서만 작업
- 자유롭게 Commit & Push 가능
- 작업 완료 후 PR을 만들어 main으로 병합 요청

---

## 브랜치 생성 명령어

```bash
# 내 브랜치로 이동 (이미 있을 때)
git checkout HyunChan

# 새로 만들 때
git checkout -b HyunChan
```

---

## 주기적으로 main 최신 내용 반영하기

다른 팀원 작업이 main에 병합되면 내 브랜치도 최신화해야 합니다.

```bash
git fetch origin
git merge origin/main
```

또는 GitHub Desktop에서 `Branch` → `Update from main` 클릭
