## 🛠️ 처음 시작하기 (최초 1회 설정)

### 1단계: 저장소 내 컴퓨터로 가져오기

**GitHub Desktop 앱 사용 시:**
1. GitHub Desktop 앱 실행
2. 상단 메뉴 `File` → `Clone Repository`
3. URL 탭에 입력: `https://github.com/touchxe/gpchurch`
4. 저장 위치 선택 후 `Clone` 클릭

**터미널 사용 시:**
```bash
git clone https://github.com/touchxe/gpchurch.git
cd gpchurch
```

### 2단계: 내 작업 브랜치 만들기

**GitHub Desktop 앱 사용 시:**
1. 상단 `Current Branch` 클릭
2. `New Branch` 클릭
3. 이름 입력 (예: `HyunChan`) → `Create Branch`

**터미널 사용 시:**
```bash
git checkout -b HyunChan   # 내 이름의 브랜치 생성
```

> ⚠️ **브랜치 이름은 항상 자기 이름으로!** 절대 main에 직접 작업하지 마세요.

## 📝 매일 작업 흐름

### STEP 1️⃣ 작업 전 — 최신 코드 받기
항상 작업 시작 전에 최신 코드를 내려받아 충돌을 예방합니다.

**GitHub Desktop 앱:**
- 상단 `Fetch origin` 클릭 → `Pull origin` 클릭

**터미널:**
```bash
git checkout main          # main으로 이동
git pull origin main       # 최신 내용 받기
git checkout HyunChan      # 내 브랜치로 돌아오기
git merge main             # main 내용을 내 브랜치에 반영
```

---

### STEP 2️⃣ 코드 수정하기
VS Code에서 파일을 열고 자유롭게 수정합니다.

> 📁 **테마 파일 위치:**
> - `wp-content/themes/gapyeong-church/` — 메인 테마
> - `wp-content/themes/gapyeong-church-child/` — 자식 테마 (주로 여기서 작업)

---

### STEP 3️⃣ 변경사항 저장 (Commit & Push)
수정이 끝나면 GitHub에 저장합니다.

**GitHub Desktop 앱:**
1. 왼쪽 패널에 변경된 파일 목록이 보임
2. 하단 `Summary` 칸에 작업 내용 간단히 입력 (예: `헤더 색상 변경`)
3. `Commit to HyunChan` 버튼 클릭
4. 상단 `Push origin` 클릭

**터미널:**
```bash
git add .                          # 변경사항 모두 선택
git commit -m "헤더 색상 변경"      # 저장 (메시지는 한국어 OK)
git push origin HyunChan           # GitHub에 올리기
```

---

### STEP 4️⃣ Pull Request (PR) 생성 — 검토 요청
작업이 완료되면 YoungBin에게 검토를 요청합니다.

1. 브라우저로 https://github.com/touchxe/gpchurch 접속
2. 상단에 노란 배너 `Compare & pull request` 버튼 클릭  
   *(없으면 `Pull requests` 탭 → `New pull request`)*
3. 제목과 설명 입력:
   - **제목:** 무엇을 바꿨는지 (예: `헤더 메뉴 디자인 수정`)
   - **설명:** 변경 이유, 확인 요청 사항 등
4. 오른쪽 `Reviewers` 에서 **`touchxe`** 선택
5. `Create pull request` 클릭

> ✅ 이 순간 YoungBin에게 자동으로 알림이 갑니다!

---

### STEP 5️⃣ 리뷰 & 수정 (필요시)
YoungBin이 코드를 보다가 수정 요청을 댓글로 남길 수 있습니다.

- 요청 사항 확인 후 로컬에서 수정
- 다시 Commit & Push하면 **같은 PR에 자동으로 반영됨**
- 별도로 새 PR을 만들 필요 없음!

## 🚀 배포 과정 (자동)

YoungBin이 Merge를 승인하면, 그 즉시 모든 것이 자동으로 이루어집니다.

```
Merge 완료
    ↓
GitHub Actions 자동 실행 (1~2분 소요)
    ↓
FTP로 Cafe24 서버에 테마 파일 자동 업로드
    ↓
https://gpchurch.mycafe24.com 에 반영 완료! 🎉
```

**배포 결과 확인 방법:**
- https://github.com/touchxe/gpchurch/actions
- ✅ 초록색 체크 = 배포 성공
- ❌ 빨간색 X = 오류 (YoungBin에게 문의)

## ⚠️ 중요한 규칙

| 규칙 | 이유 |
|------|------|
| ❌ `main` 브랜치에 직접 Push 금지 | 자동으로 라이브 서버에 배포됨 |
| ✅ 반드시 내 브랜치에서 작업 | 다른 팀원 코드와 충돌 방지 |
| ✅ 작업 전 항상 최신 코드 받기 | 최신 상태에서 시작해야 충돌 없음 |
| ✅ Commit 메시지는 명확하게 | 나중에 무슨 작업인지 파악 가능 |
| ❌ `.env`, `*.pem` 파일 절대 업로드 금지 | 서버 보안 정보 유출 위험 |
