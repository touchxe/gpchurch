# Product Requirements Document - 가평교회 홈페이지 (gpchurch)

**Author:** 현찬
**Date:** 2026-04-13

## Executive Summary

본 기획은 가평교회 성도들이 모바일(휴대폰)과 데스크톱(PC) 어떤 기기로 접속하더라도 불편함 없이 교회의 소식과 주보를 접할 수 있도록 돕는 **'반응형 크로스 플랫폼 커뮤니티 UI/UX 개편'**을 목표로 합니다. 이미지 복사, 새 창 열림이나 페이지 디자인 깨짐 등의 기존 문제들을 근본적으로 해결하여, 모든 연령대의 사용자가 직관적이고 끊김 없는(Seamless) 커뮤니티 환경을 100% 누리도록 하는 데 집중합니다.

### What Makes This Special

- **양방향 디바이스 최적화:** PC 사용자의 단축키/클릭 경험과 모바일 사용자의 스와이프/핀치 줌(확대) 제스처를 동시에 완벽히 지원하는 가볍고 안정적인 라이트박스 솔루션 도입.
- **근본적인 아키텍처 일체감:** 외부 스크립트나 CSS를 억지로 덮어씌우는 땜질 방식이 아닌, 메인 테마(`gapyeong-church-child`)와 게시판(KBoard) 시스템이 완벽하게 연동되도록 네이티브 스킨 및 CSS 변수를 활용.
- **안전하고 쉬운 관리성:** 주보 및 공지사항을 올리는 관리자의 작업 방식은 전혀 바뀌지 않으면서도, 워드프레스의 기본 메뉴(Nav Menu) 등을 활용하여 향후 다른 담당자가 훼손할 우려 없이 유지보수하기 쉬운 구조 확보.

## Success Criteria (성공 지표)

### User Success
- 성도님이 모바일에서 주보를 클릭했을 때, 별도의 창 이동 없이 즉시 고해상도 팝업이 뜨며 부드러운 확대(Pinch-zoom)가 가능함.
- 공지사항, 갤러리, 일정 등 주요 커뮤니티 페이지가 메인 홈페이지와 시각적 이질감 없이 하나의 브랜드로 느껴짐.

### Business & Admin Success
- 주간 주보 및 교회 일정에 대한 성도들의 사용법 문의가 감소함.
- 관리자가 게시물을 업로드하는 기존 방식에 변화를 주지 않고도 개선된 UI가 자동 적용되어 업무 부담이 늘지 않음.

### Technical Success
- 모든 UI 개선 사항이 모바일(아이폰/안드로이드)과 PC(Chrome/Safari 등) 환경에서 깨짐 없이 반응형으로 작동함.
- 나중에 주보 시스템을 고도화할 때 활용할 수 있도록 주보 게시판에 데이터 식별자(ID) 구조를 미리 설계함.

## Product Scope (개발 범위)

### MVP - Minimum Viable Product (현재 작업)
- [ ] 주보 게시판: 새 창 열림 방지 및 모바일 특화 라이트박스(팝업) 적용. 
- [ ] 네비게이션: 서브메뉴(주보, 교회일정) 추가 및 사이트 통합 디자인 적용.
- [ ] 게시판 통일: 공지사항, 갤러리, 일정 게시판 스킨을 메인 테마와 일치시키는 CSS 리팩토링.

### Growth Features (중기 비전)
- [ ] 홈페이지 내 주보 테마 제작: 관리자가 웹 에디터 상에서 주보의 레이아웃을 직접 선택하거나 수정할 수 있는 전용 스킨 개발.

### Vision (미래 - 최종 목표)
- [ ] 데이터 연동 자동화: 사업계획서 DB와 웹 주보 시스템을 직접 연결하여, 버튼 하나로 주보 내용이 자동 채워지고 배포되는 완전 자동화 워크플로우 구축.

## User Journeys (유저 여정)

### 1. 성도님(사용자) - 주보 확인 여정
- 홈페이지 접속 ➡️ 커뮤니티/주보 메뉴 클릭 ➡️ 최신 주보 게시물 클릭 ➡️ **라이트박스 팝업**으로 주보 이미지 노출 ➡️ 스마트폰으로 핀치 줌 확대하여 내용 확인 ➡️ 닫기 버튼으로 부드럽게 복귀.

### 2. 성도님(사용자) - 교회 일정 확인 여정
- 홈페이지 접속 ➡️ 서브메뉴에서 '교회 일정' 즉시 클릭 ➡️ **홈페이지 테마와 통일된 디자인**의 일정 게시판 확인 ➡️ 이번 달 행사 일정 체크.

### 3. 관리자(간사님) - 업로드 업무 여정
- 관리자 페이지 접속 ➡️ KBoard 글쓰기 ➡️ 이미지/파일 첨부 및 저장 ➡️ 기존과 동일한 작업 방식으로 스트레스 없이 업로드 완료 ➡️ 프론트엔드에서 자동 최적화 반영.

## Product Requirements (제품 요구사항)

### Functional Requirements (기능적 요구사항)

| ID | 요구사항 명칭 | 상세 내용 | 우선순위 |
|:---:|:---|:---|:---:|
| FR-01 | 주보 라이트박스 뷰어 | 주보 썸네일 클릭 시 전체 화면 라이트박스 오픈. 모바일 핀치 줌 확대와 PC ESC 닫기 기능 필수 지원. 뒤로 가기 시 모달만 닫히고 사이트는 유지. | High |
| FR-02 | 서브메뉴(Nav) 확장 | 커뮤니티 하위 메뉴에 '주보', '교회 일정' 링크 추가. 기존 테마 스타일과 100% 디자인 통일. 워드프레스 기본 메뉴 시스템 활용. | High |
| FR-03 | 커뮤니티 UI 스킨 통일 | 공지, 갤러리, 일정 게시판의 CSS(폰트, 컬러, 버튼, 여백)를 홈페이지 메인 테마와 일치하도록 리팩토링. 스코프 제한 필수. | Medium |
| FR-04 | 주보 데이터 식별자 | 추후 DB 연동을 위해 주보 업로드 시 '날짜/호수' 메타데이터 필드 구조 설계. | Low |

### Non-Functional Requirements (비기능적 요구사항)
- **성능:** 라이트박스 스크립트는 주보 페이지에서만 조건부 로드.
- **호환성:** iOS Safari, 안드로이드 Chrome, PC Chrome/Edge 모두 동일한 반응형 동작 보장.
- **안전성:** 자식 테마(`gapyeong-church-child`) 내에서만 수정. CSS는 스코프 제한 필수. 전역 스타일 오염 금지.
- **관리성:** 관리자 업로드 방식 변경 없이 자동 적용.
